## 0. EXECUTIVE SUMMARY

This document is the complete brand operating system for **Axial Labs**. It is not a moodboard. It is an executable, production-grade plan designed to be run phase-by-phase against Claude Sonnet to generate every asset the company will ever need — logos, tokens, mockups, social posts, decks, illustrations, motion, microsites — with consistency at scale.

**Brand thesis (one line):**

> Axial Labs builds the precision systems behind ambitious digital products.

**Visual North Star:** _A research lab that ships._ Picture Bauhaus discipline meeting Linear-grade UI craft, Apple-restraint typography, and Studio Dumbar geometric energy. The brand looks like calibrated instrumentation, not corporate software.

**Operating principle:** systems over assets. Every output we generate must be reusable, tokenized, and grid-bound. We do not produce one-off art.

**Roles in execution:**

- **Opus** = architect. Strategy, naming, tokens, prompt templates, QA.
- **Sonnet** = executor. Visual production at volume — SVG, HTML, social, illustrations, mockups.

## 1. BRAND STRATEGY

Axial Labs designs and engineers the digital infrastructure — software, brand systems, automation — that lets ambitious teams move with precision.

### 1.2 Positioning

A hybrid practice: half engineering studio, half design studio. We sit on the _axis_ between visual identity and technical execution. Where most agencies hand off PDFs and call it done, **we ship the system that runs the brand.**

**Category:** _Engineered identity studio._ (Owned, defensible, intentionally narrow.)

1. We design the system **and** the software.

## 2. NAMING ANALYSIS

### 2.1 Interpretation of "Axial Labs"

- **Axial**: relating to an axis. The imaginary line a body rotates around. Connotes: center, structure, alignment, geometry, instrumentation, mathematical rigor, engineering drawings.
- **Labs**: research, experimentation, iteration. Pre-product, prototyping, scientific method.
- **Combined meaning**: _the axis where ideas become products through experimentation._

### 2.2 Semantic opportunities

Aligned · rotation · vector · coordinate · pivot · datum · instrumentation · calibration · orbit · perpendicular · centerline · axis-aligned · canonical.

### 2.3 Symbolic concepts (visual hooks)

| Symbol    | Meaning                        | Use                            |
| --------- | ------------------------------ | ------------------------------ |
| `\|`      | the axis itself                | foundational mark              |
| `+` `×`   | intersection of axes           | system / icon                  |
| `↻`       | rotation, iteration            | motion language                |
| `/`       | vector, momentum               | accent ornament                |
| X / Y / Z | full dimensionality            | 3D illustration system         |
| `·`       | a point on an axis             | callouts, status, registration |
| ⊙         | concentric (orbit around axis) | secondary marks                |

### 2.4 Geometric meaning system

Right angles only for primary structure. Diagonals at 45°. Subtle anti-symmetry as a signature (one tick longer than the others, one bar offset by 1px). Engineering-drawing aesthetics: tick marks, dimensions, datums. Number **90°** as a conceptual anchor across the brand.

### 2.5 Visual metaphors (recurring throughout the system)

1. **The axis line** — a single vertical line as foundational mark, runs through layouts.
2. **Construction visible** — leave faint guides showing how things were built.
3. **Tick marks** — engineering measurement marks every 8/16px as recurring detail.
4. **Crosshairs / registration marks** at frame corners and intersections.
5. **Rotation as motion** — primary motion language. Not slides, not fades.

### 2.6 Sub-brand naming (future architecture)

- **Axial Studio** — client services arm
- **Axial OS** — internal templates / starter kits
- **Axial Foundry** — open source tooling
- **Axial Index** — case studies / writing
- **Axial Field** — research / experiments

## 3. LOGO SYSTEM

### 3.1 Concept direction (chosen)

**"The Axis."** A single vertical line is the foundational mark. The wordmark is built off that axis. The icon is the perpendicular intersection of two axes. This gives us a system, not a logo.

### 3.2 Primary logo — wordmark

The canonical wordmark is "AXIAL LABS" set above a horizontal measurement rule with major and minor engineering tick marks. The rule signals precision, calibration, and the brand's instrumentation aesthetic without overwhelming the type. It externalizes what the brand is: an axis, calibrated, measured.

- Type: **Inter Display** (or Neue Haas Grotesk Display 65 Medium when licensed). Weight 600. Letter-spacing −0.02em.
- viewBox `0 0 800 240`. Wordmark baseline `y=140`, centered at `x=400` with `text-anchor="middle"`.
- Measurement rule: 1px horizontal line from `x=120` to `x=680` at `y=180`. Major ticks (8u length) at `x=120, 400, 680`. Minor ticks (4u length, stroke 0.75) at `x=200, 280, 320, 480, 560, 600`.
- Rule and ticks rendered in `currentColor` at `stroke-opacity="0.4"` so the wordmark adapts to any surface without a separate file per theme.
- The vertical bar of the **A** in AXIAL still doubles as the brand's symbolic axis.
- Filenames:
  - `logos/primary/axial-wordmark-primary.svg` — currentColor (adapts to context)
  - `logos/primary/axial-wordmark-light.svg` — explicit `#0A0A0B` (use on light surfaces)
  - `logos/primary/axial-wordmark-dark.svg` — explicit `#F4F2EC` (use on dark surfaces)
  - `logos/primary/axial-wordmark-compact.svg` — bare wordmark, no rule, viewBox `0 0 600 120` (used inside lockups and at small sizes where the rule wouldn't read)
  - `logos/primary/axial-wordmark-detailed.svg` — extended lockup with left axis line + Mono index tag (`→ AL / WORDMARK / v1.0`) + Mono metadata caption (`ENGINEERED IDENTITY  ·  EST. 2026  ·  AXIS-ALIGNED`). For hero moments, manifesto/About headers, and formal collateral.

### 3.3 Secondary logo — stacked lockup

- Icon mark above wordmark, both centered on the shared axis.
- Used for: avatars, square footers, app store icons.
- Filename: `logos/lockups/axial-lockup-stacked.svg`

### 3.4 Icon mark

- A perpendicular cross (`+`) constructed from two **4u-wide bars** on a 24u grid.
- The vertical bar is **2u longer** than the horizontal — deliberate anti-symmetry, our signature.
- Becomes favicon, app icon, and OG-image cap.
- Filename: `logos/icons/axial-icon.svg`

### 3.5 Monogram

The "AL" monogram is treated as a measured datum. The letterforms sit centered in a 240×240 frame, with four small registration crosshairs at the corners — making the monogram read like a piece of engineering instrumentation rather than a styled letter pair.

- Type: **Inter Display**, weight 600, font-size 144, letter-spacing −0.02em.
- viewBox `0 0 240 240`. AL baseline `y=170`, centered at `x=120` with `text-anchor="middle"`.
- Registration crosshairs at frame corners: 8u arms crossing perpendicularly at center points (16, 16), (224, 16), (16, 224), (224, 224). Stroke `currentColor` at `stroke-opacity="0.5"`, stroke-width 1, square caps.
- The crosshairs adapt to any surface via currentColor — no separate light/dark files needed.
- Used for: partnerships, certificates, watermarks, app icons (paired with the icon mark when needed), small-format identity moments.
- Filename: `logos/monogram/axial-monogram-AL.svg`

### 3.6 Wordmark (compact, text-only)

- The bare "AXIAL LABS" with no measurement rule. viewBox `0 0 600 120`.
- Used inline in body copy, footers, document headers, **and inside the icon+wordmark lockups** (where the icon mark already supplies visual weight; the tick rule would compete).
- Used at small sizes (≤ 256px wide) where the primary wordmark's rule wouldn't render legibly.
- Filename: `logos/primary/axial-wordmark-compact.svg`

### 3.7 Responsive variants

| Size          | Variant                          | Notes                                           |

| > 1024px      | Detailed wordmark (with metadata)| Hero / manifesto / About headers                |
| 512–1024      | Primary wordmark (with tick rule)| Default solo wordmark                           |

- **Clear space**: 1× cap-height on all sides.
- **Minimum size**: wordmark 80px wide, icon 12px.

## 4. TYPOGRAPHY SYSTEM

### 4.1 Type stack

| Role          | Family                    | Fallback                      | Use                                    |
| ------------- | ------------------------- | ----------------------------- | -------------------------------------- |
| **Display**   | Neue Haas Grotesk Display | Inter Display                 | All H1/H2 above 32px                   |
| **Body**      | Inter                     | system-ui                     | 14–22px UI and prose                   |
| **Editorial** | Source Serif 4            | Tiempos Text                  | Long-form essays only                  |
| **Mono**      | JetBrains Mono            | Berkeley Mono / IBM Plex Mono | Code, data, captions, labels, eyebrows |

> **Free-stack alternative** (zero licensing): Inter Display / Inter / Source Serif 4 / JetBrains Mono. All Google Fonts. The system is designed to look identical with this stack.

### 4.2 Pairing rules

- Display + Body is the default everywhere (Neue Haas + Inter).
- Editorial serif appears **only** in long-form prose (>200 words).
- Mono is **always** lowercase or ALL CAPS — never sentence case in mono.
- Never combine more than 2 type families per layout.

### 4.3 Type scale (modular, ratio 1.25)

`12 · 14 · 16 · 20 · 24 · 32 · 40 · 56 · 72 · 96 · 128 · 160`

### 4.4 Hierarchy rules

| Style   | Family  | Size   | Weight | Tracking      | Use                                |
| ------- | ------- | ------ | ------ | ------------- | ---------------------------------- |
| H1      | Display | 72–128 | 600    | −2%           | Hero headlines                     |
| H2      | Display | 40–56  | 500    | −1%           | Section headlines                  |
| H3      | Display | 24–32  | 500    | 0%            | Subsections                        |
| H4      | Body    | 20     | 600    | 0%            | Card titles                        |
| Body L  | Body    | 18–22  | 400    | 0%            | Marketing prose                    |
| Body    | Body    | 16     | 400    | 0%            | UI text                            |
| Caption | Mono    | 12–14  | 400    | +2%           | Labels, metadata                   |
| Eyebrow | Mono    | 12     | 500    | +6% UPPERCASE | Section tags, prefixed `→` or `//` |

### 4.5 Vertical rhythm / spacing

Base unit = 4px. Type spacing: `4 · 8 · 12 · 16 · 24 · 32 · 48 · 64 · 96 · 128 · 192`.

### 4.6 Forbidden combinations

- Mono in long-form prose.
- Display below 24px.
- Italics outside of citations.
- Justified text (always left-align or set to grid).
- Underlines on links — use Voltage chip or 1px Voltage underline only.

| `text.secondary` | `#B5B9C2` | `#5A5E68`                        |
| `border`         | `#26282D` | `#D8DAE0`                        |

### 5.8 Accessibility rules

- Body text: **AA minimum (4.5:1)**, **AAA (7:1)** for editorial long-form.
- **Voltage is a signal color, not a text color.** It must always sit on Axial Black. For Voltage on Bone, use `voltage.dim`.
- Focus rings: 2px Voltage offset 2px from element.
- Never rely on color alone — pair with iconography or text.

### 5.9 Use ratios (rule of thumb)

**60% Axial Black or Bone · 35% neutrals · 5% Voltage.**
Voltage is a punctuation mark, not a paint. If a layout looks "too green," it is.

## 6. VISUAL LANGUAGE

### 6.1 Shape language

- Right angles only for primary structure. **No rounded cards by default.**
- Subtle 4px corner radius reserved for inputs and chips.
- Circles only as functional markers (status dots, focus rings, registration marks).
- Diagonals at **45° only** — never arbitrary angles.

### 6.2 Grid system

- Marketing pages: **12-column**, 24px gutter, 1280px max-width.
- Product UI: **8-column**, 16px gutter.
- 4px base unit governs everything.
- A **persistent visible grid** is part of the brand — show 1px graphite-200 grid lines in some hero moments. Construction is on display.

### 6.3 Iconography

- Style: **1.5px stroke, square caps, 24×24 viewbox, no fills.**
- All icons constructed on a visible underlying grid.
- Library: custom, seeded from Lucide / Phosphor at start.
- Naming: `icon-axis-vertical.svg`, `icon-pivot.svg`, `icon-orbit.svg`, etc.
- Color: `currentColor` so they inherit from context.

### 6.4 Illustrations

- **Technical-drawing style.** Engineering schematics, blueprint marks, ISO drawing conventions.
- Color: monochrome graphite + Voltage highlights only.
- Built from the same 4px grid as everything else.
- SVG-only, with semantic classes: `.axis-line`, `.tick`, `.datum`, `.callout`.
- Always include at least 2 measurement tick marks somewhere.

### 6.5 Motion language

- **Easing**: `cubic-bezier(0.32, 0.72, 0, 1)` — the custom "Axial" curve.
- **Durations**: 240ms (UI), 600ms (hero), 1200ms (orchestrated entries).
- **Primitives**: rotation around an axis · axis-line wipes · character-by-character type reveals · parallax along Y · ticker-style mono labels.
- **Forbidden**: bounces, elastic, anything cute, drop-shadow morphs, 3D flips.

### 6.6 Texture

- Subtle film grain on hero backgrounds (3–5% opacity).
- Occasional 1px scanline overlays.
- Engineering-paper backgrounds for case study sections (very subtle 0.5px grid).

### 6.7 Patterns (SVG-defined, themable)

- Tick-mark borders (8 or 16px increments).
- Dotted axis guides.
- Crosshair registration marks at frame corners.
- 1px grid overlay (3% opacity).

### 6.8 Lighting

- "**Lab lighting**" — flat, even, shadow-free for product shots and UI.
- One dramatic accent glow (Voltage) for hero moments.
- Avoid: photorealistic shadows, drop shadows on flat UI, ambient gradients in product UI.

### 6.9 Depth system (Z-axis)

| Layer | Use                    | Spec                                                           |
| ----- | ---------------------- | -------------------------------------------------------------- |
| Z0    | Page surface           | No shadow                                                      |
| Z1    | Cards / elevated       | 1px border; **no shadow** in dark mode; subtle shadow in light |
| Z2    | Dropdowns / popovers   | Subtle shadow + 1px border                                     |
| Z3    | Modals                 | Backdrop blur 12px + scrim `rgba(10,10,11,0.6)`                |
| Z4    | Toasts / notifications | Voltage left border (3px), no shadow                           |

**The system prefers borders over shadows. Linear-style.**

### 6.10 Reference set (mood)

**Product UI**: Linear · Vercel · Arc · Stripe (post-2023) · Raycast · Cursor · Resend · Granola · Cron · Things 3 · Loops.so · Anthropic.com · Bun.sh · Tailwind UI Catalyst.
**Art direction / studios**: Studio Dumbar · Pentagram · Bureau Borsche · Manual · Daybreak · Order · Workshop.
**Typography**: Klim Type Foundry · Pangram Pangram · Lineto · Grilli Type.

### 7.5 Service cards

- 4-up grid. Square cards. **No rounded corners.**
- Each card: number (`01 / 04`), title (H4), 1-line description, bottom-right `→` arrow.
- Hover: card border becomes Voltage; arrow translates 4px right.
- Cards never use background fills — they use **borders and white space**.

### 7.6 Navigation

- Slim 56px top bar, fixed.
- Left: icon mark + wordmark
- Center: nav links in Mono UPPERCASE 12px tracking +6%
- Right: contact CTA in Voltage outline button

### 7.7 Footer system

Three sections, vertically stacked:

1. Massive type **AXIAL LABS** wordmark, full-width, edge-to-edge.
2. Three-column links (Services · Work · Writing · Contact · Locations).
3. Mono footer line: `© 2026 — AXIAL LABS  //  SYSTEMS FOR AMBITIOUS SOFTWARE`

### 7.8 Responsive behavior

- Breakpoints: **480 / 768 / 1024 / 1440**.
- Type fluid via `clamp()`.
- Grid collapses 12 → 8 → 6 → 4 columns.
- Massive type stays massive on mobile (do **not** shrink the footer wordmark).

### 8.4 SVG illustration style for social

Always: 1.5px strokes · no fills (or single Voltage fill) · 2 measurement tick marks · sit on a visible axis line · monochrome graphite + Voltage only.

1. Includes the icon mark **or** wordmark.
2. Includes the Mono index/tag in either top-left or bottom-right.
3. 24px outer margin minimum.
4. 1 surface + 1 accent + max 1 supporting tone.
5. Never combines more than 2 type styles per slide.

## 11. AI EXECUTION WORKFLOW

### 11.1 Roles

- **Opus** = the architect. Strategy, voice, naming rationale, taxonomy, design tokens, prompt templates, system documentation, review/QA, brand guidelines master.
- **Sonnet** = the executor. Visual asset generation: SVG, HTML, CSS, Instagram templates, illustrations, motion specs, mockups in volume.

### 11.2 Order of generation (canonical)

1. _(Opus)_ Strategy + voice + naming → `/branding/*.md`
2. _(Opus)_ Design tokens → `/tokens/**/*.json`
3. _(Opus)_ System prompt + reusable prompt templates → `/prompts/*.md`
4. _(Sonnet)_ Logo system SVGs (using Opus's construction rules)
5. _(Opus)_ Logo QA — review proportions, regenerate where needed
6. _(Sonnet)_ Color palette HTML, type specimen HTML, gradient SVGs
7. _(Sonnet)_ Icon set (16-icon starter pack)
8. _(Sonnet)_ Illustration starter pack (8 illustrations)
9. _(Sonnet)_ HTML mockups in priority order: landing → pricing → services → case study → about
10. _(Sonnet)_ Instagram templates (5 base templates + 1 carousel)
11. _(Opus)_ Brand guidelines master doc → `/branding/06-brand-guidelines.md`
12. _(Sonnet)_ Pitch deck template
13. _(Opus + Sonnet)_ Iterate

### 11.3 Maintaining consistency across generations

- Every prompt to Sonnet **starts with** the canonical system prompt loaded from `/prompts/system-prompt.md`.
- That system prompt declares: tokens, type rules, color use ratios, forbidden patterns, file naming.
- Every generated asset is reviewed against the **12-item QA checklist** (§13) before saving.
- Tokens are **never** inlined as raw values — always reference by name (`var(--axial-voltage)`).
- Every HTML mockup imports `tokens.css` and the Tailwind config snippet, never duplicates values.

### 11.4 Versioning

- Brand system follows **semver**: `v1.0.0 = first complete release`.
- Token changes that break consumers = major bump.
- Net-new tokens or assets = minor bump.
- Visual fixes / typo corrections = patch.

## 12. PROMPT SYSTEM

### 12.1 The canonical system prompt — `/prompts/system-prompt.md`

This block is **prepended to every Sonnet call** that produces a brand asset.

```md
You are generating an asset for **Axial Labs**, an engineering-and-design studio.

# Hard rules (non-negotiable)

- Color: only use the Axial palette. Tokens, in order of priority:
  --axial-black #0A0A0B · --axial-bone #F4F2EC · --axial-voltage #C6F24E
  graphite ramp 100..900 · accents (ion #3D5BFF, plasma #FF4D2E, mercury #B8C2CC) used rarely
- Voltage may cover ≤ 5% of pixels in any composition. It is a signal, not a paint.
- Type families: Display = Inter Display / Neue Haas Grotesk Display.
  Body = Inter. Editorial = Source Serif 4. Mono = JetBrains Mono.
- Mono is always lowercase or ALL CAPS, never sentence case.
- Right angles only for primary structure. Diagonals only at 45°.
- Grid: 4px base unit. Everything aligns to it.
- No drop shadows on flat UI. Prefer 1px borders.
- No bouncy / elastic motion. Use ease cubic-bezier(0.32, 0.72, 0, 1).
- SVG: 1.5px stroke, square caps, viewBox set, no fills unless using Voltage.
- Always include at least one "axial" element: an axis line, tick marks,
  a crosshair, a registration mark, or a Mono index label like "01 / 04".
- Never use generic stock-illustration tropes (rocketships, gears, lightbulbs).

# Output format

Return a single self-contained file. No commentary outside the file.
Use the exact filename specified in the user message.

# Voice (when generating copy)

- Few words. Specifics over slogans. No adjectives unless necessary.
- Forbidden: synergy, solutions, innovative, cutting-edge, game-changing,
  empower, unlock, leverage, seamless, robust, world-class.

# Failure modes to avoid

- Drifting toward "generic SaaS" rounded gradient cards.
- Adding emoji.
- Using more than 2 type families in one composition.
- Centering everything (we left-align by default).
- Using Voltage as text on Bone (use voltage-dim instead).
```

## APPENDIX A — FONT LICENSING NOTES

- Inter, Inter Display, Source Serif 4, JetBrains Mono: free (OFL).
- Neue Haas Grotesk Display: paid (Linotype / Monotype). Recommended once revenue justifies; Inter Display covers ~95% of the visual feel.
- Berkeley Mono: paid (Berkeley Graphics). Optional upgrade over JetBrains Mono.

## APPENDIX B — DEFAULT BREAKPOINTS

`xs: 0` · `sm: 480` · `md: 768` · `lg: 1024` · `xl: 1280` · `2xl: 1440`

## APPENDIX C — RESERVED TERMS (always set in Mono)

`AXIAL` · `LABS` · `OS` · `STUDIO` · `FOUNDRY` · `INDEX` · `FIELD` · all version numbers · all coordinates · all timestamps.

## APPENDIX D — ANTI-PATTERNS (never ship these)

- Rocket-ship illustrations.
- Gradient blob backgrounds.
- Neumorphism.
- Glassmorphism with heavy blur.
- 3D isometric icons.
- Emoji in product UI.
- "Lightning bolt" anywhere.
- Center-stage marketing layouts ("hero with subhead and one CTA centered").
- Drop shadows on cards in dark mode.
- Color-coded charts with > 4 hues.
